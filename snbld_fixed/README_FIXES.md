# 🔧 Исправления критических проблем в логике макросов

## ✅ Исправленные проблемы

### 1. Удалена дублирующая установка `cast_lock_until`
**Проблема:** Блокировка каста устанавливалась дважды:
- В `macros_dispatcher.request_macro()` (правильно)
- В `macros_core._run()` после шага 1 (дубль, создавал гонку)

**Решение:** Удалена вторая установка в `macros_core.py` (строки 902-910 и 1010-1016).
Теперь блокировка устанавливается ТОЛЬКО в диспетчере атомарно перед запуском макроса.

### 2. Удалена тройная запись `last_used`
**Проблема:** Переменная `last_used` записывалась в трёх местах:
- При обнаружении кастбара (строка 961)
- В режиме без детекции кастбара (строка 988)
- В конце `_run()` (строка 1058)

Это сбивало кулдауны - макросы запускались чаще положенного.

**Решение:** Удалены все промежуточные записи. Теперь `last_used` обновляется 
ТОЛЬКО в конце `_run()` после полного завершения макроса (строка 1059).

### 3. Исправлен сброс `cast_lock` в `on_macro_finished`
**Проблема:** Метод использовал `cast_lock_margin` как задержку ПОСЛЕ завершения макроса,
создавая "окно уязвимости" когда другие макросы не могли запуститься.

**Решение:** Создан новый метод `on_macro_finished_immediate()` который сбрасывает 
блокировку НЕМЕДЛЕННО после завершения макроса. `cast_lock_margin` используется 
ТОЛЬКО для установки блокировки ПЕРЕД кастом, а не для задержки сброса ПОСЛЕ.

### 4. Исправлена логика детекции кастбара
**Проблема:** Если кастбар не найден, макрос прерывался и шаг 3 (ресвап) не выполнялся.
Игрок оставался в уязвимом сете для пения!

**Решение:** Теперь даже если кастбар не обнаружен, макрос продолжает выполнение 
и выполняет шаг 3 (ресвап на ПА). Это предотвращает ситуацию когда игрок остаётся 
без защитного сета.

## 🚀 Как использовать

1. **Сохраните текущую версию** (на всякий случай):
   ```bash
   cp -r macros_core.py macros_core_backup.py
   cp -r backend/macros_dispatcher.py backend/macros_dispatcher_backup.py
   ```

2. **Замените файлы**:
   ```bash
   cp snbld_fixed/macros_core.py .
   cp snbld_fixed/backend/macros_dispatcher.py backend/
   ```

3. **Запустите программу** и проверьте работу макросов.

## 📊 Ожидаемые улучшения

- ✅ Стабильные кулдауны - макросы не запускаются чаще положенного
- ✅ Правильная блокировка каста - нет гонок между макросами
- ✅ Надёжный ресвап - всегда выполняется шаг 3 даже без кастбара
- ✅ Быстрая разблокировка - следующие макросы запускаются сразу после завершения текущего

## ⚠️ Важные замечания

- `cast_lock_margin` теперь используется ТОЛЬКО для установки блокировки ПЕРЕД кастом
- Не изменяйте значение `cast_lock_margin` без понимания последствий
- Логика ресвапа до окончания каста сохранена полностью
